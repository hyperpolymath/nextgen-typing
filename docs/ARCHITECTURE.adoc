== What nextgen-typing Is

`nextgen-typing` is a coordination monorepo that documents and connects
the hyperpolymath type theory pipeline.  It does not contain compiler
code.  It contains:

* Architecture docs explaining the main chain and co-products
* Roadmap and status tracking
* Cross-repo integration notes
* Research artefacts that don't belong in a single language repo

It follows the pattern of `nextgen-languages` and `nextgen-databases`.

== The Main Chain

[source]
----
katagoria  →  typell  →  typed-wasm  →  PanLL
  (research)   (kernel)   (target)    (eNSAID env)
----

[cols="1,1,2"]
|===
| Repo | Status | Role

| `katagoria`
| ✅ Active (created 2026-04-11)
| Research origination.  Where type theory ideas are explored before
  being promoted to a production repo.  Contains proofs-of-concept,
  reading notes, and prototype Idris2/Lean formalizations.

| `typell`
| ✅ Active
| The TypeLL verification kernel.  *Open-ended progressive* type safety
  framework — NOT capped at any fixed number of levels.  Powers PanLL's
  neurosymbolic intelligence.  Runs as a service on port 7800 in PanLL.
  14 Rust crates, 13 language bridge crates.

| `typed-wasm`
| ✅ Active
| Dual role (ADR-004): (1) TypeLL type safety for WasmGC linear memory
  (current checked core: L1-L10), (2) aggregate library — formally
  verified binary conventions for AffineScript and Ephapax cross-language
  calls.  No FFI layer needed when both languages agree on layout.

| `PanLL`
| ✅ Active
| eNSAID reference implementation.  TypeLL runs inside PanLL via the
  JSON-RPC verification protocol (port 7800).  Not owned by nextgen-typing —
  the downstream consumer.
|===

=== Key Invariant: TypeLL Is Open-Ended

TypeLL is NOT a "10-level type safety system."  It is an open-ended
progressive framework.  The L1-L10 mapping is the initial implementation,
used by the VCL-total bridge and applied to WasmGC memory safety by
typed-wasm.  New levels are added as the type theory demands.  There is
no ceiling.

This distinction matters for documentation and for academic publications:
the research contribution is *progressive* type safety, not a fixed
10-level hierarchy.

== Co-Products

These repos benefit from or contribute to the typing pipeline but are not
part of the main chain:

[cols="1,2"]
|===
| Repo | Relationship

| `nextgen-languages/affinescript`
| Compiles to typed WasmGC.  Row polymorphism, algebraic effects, and QTT
  surface syntax are outcomes of TypeLL-grounded design.  Uses typed-wasm
  as aggregate library for binary layout conventions.

| `nextgen-languages/ephapax`
| Compiles to typed WasmGC.  Dyadic (linear/affine) type system is a direct
  application of TypeLL principles.  Uses typed-wasm as aggregate library.

| `protocol-squisher`
| The 4-class transport semilattice (Concorde/Business/Economy/Wheelbarrow)
  is a tropical optimisation problem.  Minimax adapter path search is
  Dijkstra over a max-plus semiring — formally a tropical computation.
  The `proofs/tropical/` subtree documents the formal connection.
|===
